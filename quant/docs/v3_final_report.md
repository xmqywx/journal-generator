## 🔧 技术架构

### 1. MarketDetectorV2

**核心改进：**
- 多时间周期：180/150/90/30天（减少365天滞后）
- 趋势减速检测：扣分-3.0（识别假突破）
- 从高点回撤检测：扣分-3.0（避免高位买入）
- 趋势反转检测：±2.0（早期识别反转）

**评分体系：**
```python
基础得分：
  180天趋势：-2.5到+2.5
  150天趋势：-2.5到+2.5
  90天趋势：-2.5到+2.5
  30天趋势：-2.0到+2.0

调整因子：
  趋势减速：-3.0到0
  高点回撤：-3.0到0
  趋势反转：-2.0到+2.0

总分归一化：(score + 14) / 2.6 → 0-10分
```

**趋势强度分级：**
| 评分 | 强度 | 操作 |
|------|------|------|
| 8.0+ | STRONG_BULL | 满仓 |
| 6.5-7.9 | MODERATE_BULL | 可持有 |
| 5.5-6.4 | WEAK_BULL | 观望 |
| 4.5-5.4 | RANGING | 空仓 |
| 3.0-4.4 | WEAK_BEAR | 空仓 |
| <3.0 | STRONG_BEAR | 空仓 |

### 2. 交易策略（滞后性机制）

**买入条件（严格）：**
```python
if current_state != 'BULL':
    # 评分 > 7.5
    # AND 减速扣分 > -2.0（无明显减速）
    # AND 回撤扣分 > -2.0（无大幅回撤）
    if score > 7.5 and decel > -2.0 and drawdown > -2.0:
        BUY with 2x leverage
```

**卖出条件（果断）：**
```python
if current_state == 'BULL':
    # 评分 < 4.0（熊市或深度震荡）
    if score < 4.0:
        SELL and go to cash
```

**滞后性优势：**
- 避免在边界频繁切换（5.0-6.5震荡区）
- 只在强确认信号时买入（>7.5 + 无减速/回撤）
- 及时止损（<4.0）

### 3. 杠杆模式（保证金）

**正确的2x杠杆计算：**
```python
开仓：
  position_value = capital * leverage
  commission = position_value * fee_rate
  capital -= commission

平仓：
  price_change% = (exit_price / entry_price - 1)
  pnl = capital * price_change% * leverage
  commission = capital * leverage * (exit_price/entry_price) * fee_rate
  capital += pnl - commission
```

**不是**简单的 `capital * 2` 买入BTC！

## 🎓 核心经验总结

### 1. 多时间周期的重要性

**错误：** 单一60天周期 → 46%准确率
**正确：** 180/150/90/30天组合 → 62.9%牛市识别率

**原理：**
- 长期趋势（180天）：判断大方向
- 中期趋势（90天）：确认趋势健康
- 短期趋势（30天）：检测反转信号

### 2. 减速检测是关键

**发现：**
- 评分7.71 + 减速-3.0 = 假突破（牛市陷阱）
- 评分6.70 + 减速-0.5 = 真牛市

**教训：**
表面评分高不够，必须检查趋势是否加速还是减速。

### 3. 滞后性避免频繁交易

**错误：** 评分5.0-6.5频繁切换 → 179笔交易
**正确：** 买入>7.5，卖出<4.0 → 6笔交易

**原理：**
在边界区（4.0-7.5）保持当前状态，只在强确认时切换。

### 4. 杠杆不是本金翻倍

**错误理解：** 2x杠杆 = 用20,000买BTC
**正确理解：** 2x杠杆 = 用10,000保证金，盈亏按2x计算

**影响：**
错误计算会导致收益虚高8000%+，完全脱离现实。

### 5. 宁可错过，不可做错

**统计：**
- 避免2笔坏交易（-40%，-21%）
- 总计避免亏损-61%
- 抵得上1.5笔好交易的盈利

**结论：**
严格的买入条件（7.5+减速+回撤）比宽松的条件更赚钱。

## 📉 风险与局限性

### 1. 样本量小（3笔交易）

**风险：** 可能过拟合历史数据
**缓解：**
- 使用简单规则（7.5阈值），不是复杂模型
- 规则基于市场逻辑（减速=放缓，回撤=高位）
- 未来需要实盘验证

### 2. 未考虑极端情况

**风险：** 黑天鹅事件（闪崩、交易所关闭）
**缓解：**
- 使用止损（评分<4.0）
- 避免满仓（虽然当前是满仓2x）
- 建议实盘降低杠杆到1.5x或1x

### 3. 熊市空仓机会成本

**问题：** 熊市期长期空仓，错过反弹
**表现：**
- 2025-2026熊市期，v3系统获利+93.75%（2笔交易）
- 说明反而是机会，不是问题 ✅

### 4. 依赖历史规律

**假设：** 未来市场会延续历史模式（牛熊周期）
**风险：** 如果市场结构变化（如监管、大机构介入），策略可能失效
**建议：** 定期回顾（每季度），必要时调整参数
