# v3量化系统 - 增强版Web UI 设计文档

**设计日期：** 2026-02-28
**状态：** 已批准

---

## 1. 目标

### 1.1 问题陈述
当前 Web UI（v1）功能基础，缺少关键功能：
- ❌ 无趋势图表，无法可视化分析市场
- ❌ 无配置界面，无法调整交易参数
- ❌ 交易历史信息单薄，无详细记录
- ❌ 无回测结果展示，缺乏策略信心参考
- ❌ 无风险提醒，安全性不足
- ❌ 无日志查看，排查问题困难

### 1.2 设计目标
创建专业级量化交易仪表板，满足实盘交易的完整需求：
- ✅ 完整图表组合（价格走势、评分趋势、多周期对比）
- ✅ 可配置交易参数（资金、杠杆、手续费）
- ✅ 交易历史详情（可点击查看完整市场状态）
- ✅ 回测结果展示（452%收益作为参考）
- ✅ 实时风险提醒（回撤、止损距离）
- ✅ 日志查看（历史每日检查记录）

### 1.3 设计原则
1. **快速决策** - 一屏看到所有关键信息（状态、图表、信号）
2. **安全第一** - 风险提醒始终可见，策略参数只读
3. **专业感** - 类似量化交易终端，非普通网站
4. **移动友好** - 响应式设计，手机端可用

## 2. 技术架构

### 2.1 技术栈

**后端（保持不变）：**
- Flask 3.0.0
- LiveTrader（市场分析和交易执行）
- MarketDetectorV2（核心策略引擎）
- BinanceFetcher（数据获取）

**前端（升级）：**
- HTML5 + Tailwind CSS 3.x（现有样式系统）
- Chart.js 3.9+（图表库，MIT许可，42KB gzipped）
- Vanilla JavaScript（无额外框架依赖）


## 5. 用户流程设计

### 5.1 首次访问流程
1. 用户访问 http://localhost:5001
2. 页面加载，显示骨架屏（loading skeleton）
3. 发起 `/api/status` 和 `/api/chart_data` 请求
4. 后端执行 LiveTrader.daily_check()（如果今天未检查）
5. 前端渲染所有组件
6. 启动30秒定时器，定期刷新

### 5.2 配置修改流程
1. 用户点击"配置管理"展开面板
2. 修改初始资金/杠杆/手续费
3. 点击"保存配置"
4. 前端发送 POST /api/update_config
5. 后端验证参数（杠杆1-3x，手续费0.02-0.1%）
6. 保存到 config.json
7. 重载 LiveTrader 实例
8. 返回成功，前端显示提示并刷新数据

### 5.3 执行交易流程
1. 用户看到"🟢 建议买入"信号
2. 点击"确认买入"按钮
3. 系统弹窗："确认执行买入操作？请确保已在Binance手动下单。"
4. 用户在 Binance 完成下单，记录成交价
5. 系统弹窗："请输入实际成交价格（当前价格: $65,000.00）："
6. 用户输入成交价（如 65,120.50）
7. 前端发送 POST /api/execute_trade
8. 后端调用 LiveTrader.execute_trade('BUY', 65120.50)
9. 更新状态，保存日志
10. 返回成功，前端刷新所有数据

### 5.4 查看交易详情流程
1. 用户在交易历史表格点击 [🔍]
2. 打开模态框，显示加载中
3. 发送 GET /api/trade_detail/1
4. 后端从日志读取该笔交易的完整信息
5. 返回详情（开仓市场状态、平仓原因、持仓期间价格波动）
6. 前端渲染模态框内容
7. 用户点击关闭按钮或背景关闭模态框

### 5.5 查看日志流程
1. 用户滚动到日志查看器
2. 默认显示最近30条每日检查记录
3. 用户选择日期范围筛选
4. 发送 GET /api/logs?start=2026-01-01&end=2026-02-28
5. 后端从 live_trading_log.json 读取符合条件的 checks
6. 返回日志列表
7. 前端渲染日志条目
8. 用户可点击展开查看完整详情

## 7. 安全性和限制

### 7.1 参数验证
**配置更新时的验证：**
- 初始资金：100 ≤ capital ≤ 100,000 USDT
- 杠杆倍数：1.0 ≤ leverage ≤ 3.0
- 手续费率：0.0002 ≤ fee_rate ≤ 0.001

**策略参数保护：**
- 买入阈值、卖出阈值、过滤器阈值、时间周期 → 只读显示
- 前端禁用输入，后端不接受修改请求

### 7.2 错误处理
**网络错误：**
- 显示友好提示："无法连接服务器，请检查网络"
- 保持当前数据显示，不清空界面
- 重试机制：3秒后自动重试

**数据获取失败：**
- 显示："市场数据获取失败，可能是Binance限流"
- 显示上次成功获取的时间
- 建议用户等待5分钟后刷新

**交易执行失败：**
- 显示具体错误原因
- 不修改持仓状态
- 记录错误到日志

### 7.3 性能优化
**图表数据量控制：**
- 价格走势图：最多显示180天数据（约4320个数据点）
- 评分趋势图：最多显示90天数据
- 数据点过多时，按天聚合（使用每日收盘价）

**缓存策略：**
- 图表数据缓存5分钟（前端本地缓存）
- 状态数据实时获取（30秒刷新）
- 历史交易和日志缓存10分钟

**懒加载：**
- 日志默认只加载最近30条
- 滚动到底部时加载更多
- 交易详情按需加载（点击时才请求）

## 8. 风险和限制

### 8.1 已知限制
1. **图表数据依赖日志** - 如果之前没有保存每日评分，图表会从今天开始记录
2. **实时性受限** - 30秒刷新间隔，非真正实时
3. **单用户设计** - 不支持多用户同时使用
4. **无身份验证** - 任何能访问 localhost:5001 的人都可操作

### 8.2 未来可扩展方向
1. **WebSocket实时推送** - 替代轮询，降低延迟
2. **移动端App** - 基于 React Native 的原生应用
3. **多交易对支持** - 不仅限于BTC，支持ETH等
4. **更多图表** - 技术指标图（RSI、MACD等）
5. **告警通知** - 邮件/短信/Telegram 通知
6. **用户认证** - 密码保护，防止未授权访问
