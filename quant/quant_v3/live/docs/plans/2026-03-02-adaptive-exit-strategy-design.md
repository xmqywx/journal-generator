## 一、问题分析

### 1.1 当前策略的核心问题

**胜率极低**：
- SOL: 7.14% (14笔交易仅1笔盈利)
- ADA: 0% (4笔全亏)

**卖出逻辑致命缺陷**：
- **SOL案例**：买入$169，最高涨到$247(+45.7%)，最终在$153卖出(-9.5%)
  - 9月17日峰值$235时不卖（盈利+38%）
  - 9月21日跌到$215时不卖（从峰值-8.5%）
  - 9月24日暴跌到$190时不卖（从峰值-19%）
  - 10月1日反弹到$220时不卖（死猫跳）
  - 10月10日触发止损在$153（历史最低点）

**三大致命缺陷**：
1. **无止盈机制**：盈利45%不卖，等着回吐
2. **卖出阈值太低**：评分5.0太低，从7.5降到5.0需要很长时间，期间价格已暴跌
3. **反弹陷阱**：评分系统滞后，无法区分"真反转"和"死猫跳"

### 1.2 用户需求

- ✅ **目标胜率**: 50%以上（稳健型）
- ✅ **交易频率**: >10笔/年（保持较高活跃度）
- ✅ **杠杆**: 维持3倍（高风险高收益）
- ✅ **通用性**: 适配所有币种

### 1.3 核心洞察

用户的关键观察：
> "买入点还可以，只是卖出点有严重的问题"
> "如图在该买的时候评分低，在改卖的时候没有及时卖出"
> "类似SOL这种大波动，应该短线包利润。BTC这种稳定波动，应该合理优化止损"

**设计方向**：
- 不是调整固定参数
- 而是根据币种波动率特性，使用不同的卖出策略
- 在买入时就判断币种类型，然后应用对应的卖出逻辑

## 二、设计方案

### 2.1 整体架构

```
┌─────────────────────────────────────────┐
│         自适应卖出策略系统              │
└─────────────────────────────────────────┘
            │
            ├─ 第一层：波动率检测（买入时）
            │   └─ 计算日波动率、周波动率、ATR、极端波动
            │   └─ 分类：STABLE / MODERATE / HIGH
            │
            ├─ 第二层：策略参数选择（买入后）
            │   ├─ STABLE型（BTC/ETH）
            │   │   └─ 宽松止盈、宽松止损、追大趋势
            │   ├─ MODERATE型（主流币）
            │   │   └─ 中等参数
            │   └─ HIGH型（SOL/山寨）
            │       └─ 严格止盈、快速锁定、短线思维
            │
            └─ 第三层：动态卖出决策（持仓中）
                ├─ 快速锁定（仅高波动型）
                ├─ 分段止盈（多次部分卖出）
                ├─ 评分卖出（动态阈值）
                └─ 止损保护（根据杠杆调整）
```

### 2.2 波动率检测系统

#### 2.2.1 检测指标

```python
class VolatilityDetector:
    """币种波动率检测器（买入时执行一次）"""

    def calculate_volatility(self, df: pd.DataFrame) -> dict:
        """
        计算多维度波动率指标

        指标：
        1. 日均波动率（过去30天）
        2. 周均波动率（过去12周）
        3. ATR百分比（过去14天）
        4. 极端波动（最大单日跌幅，过去60天）

        返回示例：
        {
            'daily_volatility': 0.035,
            'weekly_volatility': 0.082,
            'atr_percentage': 0.042,
            'max_drawdown_speed': 0.15,
            'volatility_level': 'STABLE'
        }
        """
```

#### 2.2.2 分类标准

| 波动率类型 | 日波动率 | 周波动率 | 最大单日跌幅 | 典型币种 |
|-----------|---------|---------|-------------|---------|
| STABLE | <3% | <5% | <8% | BTC, ETH |
| MODERATE | 3-5% | 5-10% | 8-15% | BNB, XRP |
| HIGH | >5% | >10% | >15% | SOL, ADA, 山寨币 |

### 2.3 分级卖出策略

#### 2.3.1 STABLE型策略（BTC/ETH）

**目标**：抓住大趋势，容忍正常回调

| 参数 | 值 | 说明 |
|-----|---|------|
| 卖出阈值 | 6.0 | 宽松（买入7.5，卖出6.0，差1.5分） |
| 止损 | 12% | 3倍杠杆下，价格波动4%触发 |
| 止盈1 | 盈利50%，回撤15% | 大盈利，允许大回调 |
| 止盈2 | 盈利30%，回撤12% | 中等盈利 |
| 分段止盈 | 启用 | 不一次性清仓 |
| 移动止损 | 启用 | 跟随趋势保护利润 |

**案例**：
```
买入 $42,000
涨到 $73,000 (+73.8%)
回调到 $65,000 (-11% from peak)
→ 回撤11% < 15%，继续持有 ✅
继续涨到 $99,000
回撤 > 15%时卖出50%
```

#### 2.3.2 MODERATE型策略（主流币）

| 参数 | 值 | 说明 |
|-----|---|------|
| 卖出阈值 | 6.3 | 中等 |
| 止损 | 11% | 中等 |
| 止盈1 | 盈利40%，回撤12% | 中等参数 |
| 止盈2 | 盈利25%，回撤10% | 中等参数 |
| 分段止盈 | 启用 | |
| 移动止损 | 启用 | |

#### 2.3.3 HIGH型策略（SOL/山寨币）

**目标**：快速锁定利润，不贪婪

| 参数 | 值 | 说明 |
|-----|---|------|
| 卖出阈值 | 6.5 | 严格（差距仅1.0分） |
| 止损 | 10% | 严格 |
| 止盈1 | 盈利35%，回撤8% | 严格止盈 |
| 止盈2 | 盈利20%，回撤10% | 严格止盈 |
| **快速锁定** | **盈利25%，卖40%** | **新增！核心特性** |
| 分段止盈 | 启用 | 多次部分卖出 |
| 移动止损 | 禁用 | 太激进 |

**快速锁定机制**（仅高波动型）：
```python
if 盈利 >= 25%:
    卖出40%仓位  # 立即锁定利润，不等回撤
    剩余60%继续持有，用更严格的止盈保护
```

**案例（修复SOL问题）**：
```
买入: $169.55

9月17日 $235 (+38.6%):
  ✅ 触发快速锁定：卖出40% @ $235
  → 锁定利润 +$524

9月21日 $215 (-8.5% from peak):
  ✅ 剩余60%仓位，盈利26.8%，回撤8.5%
  ✅ 触发止盈：盈利20%+回撤>8%
  → 再卖出30% @ $215
  → 锁定利润 +$161

累计卖出70%，总盈利 +$685
剩余30%用止损保护

最终结果：
  原策略: -$190（亏损）
  新策略: +$560（盈利）
  改进: +$750 💰
```

### 2.4 卖出决策流程

```python
class AdaptiveExitStrategy:
    """自适应卖出决策"""

    def check_exit(position_info, current_data, vol_level) -> dict:
        """
        返回: {
            'action': 'HOLD' / 'SELL_PARTIAL' / 'SELL_ALL',
            'sell_ratio': 0.4,  # 如果是部分卖出
            'reason': '原因说明'
        }
        """

        # 1. 快速锁定利润（仅HIGH型）
        if vol_level == 'HIGH':
            if 盈利 >= 25%:
                return SELL_PARTIAL(40%, '快速锁定利润')

        # 2. 分段止盈（所有类型）
        for 止盈级别 in 策略参数:
            if 盈利 >= 阈值 and 回撤 >= 容忍度:
                if 盈利 < 50%:
                    return SELL_PARTIAL(50%, '分段止盈')
                else:
                    return SELL_ALL('大盈利止盈')

        # 3. 评分卖出
        if 评分 < 卖出阈值:
            return SELL_ALL('评分恶化')

        # 4. 止损
        if 亏损 > 止损线:
            return SELL_ALL('触发止损')

        return HOLD
```

### 3.3 数据流

```
买入阶段:
  1. 评分触发买入条件
  2. VolatilityDetector.calculate_volatility(df)
  3. 分类为 STABLE/MODERATE/HIGH
  4. 记录到 position_info['vol_level']
  5. 选择对应策略参数

持仓阶段:
  每个bar:
    1. 更新峰值价格
    2. AdaptiveExitStrategy.check_exit()
    3. 根据返回结果:
       - HOLD: 继续持有
       - SELL_PARTIAL: 部分卖出
       - SELL_ALL: 全部卖出
    4. 记录交易明细（包含volatility_level）
```

## 五、风险与限制

### 5.1 潜在风险

**过早止盈风险（BTC）**：
- 问题：BTC大牛市可能涨几个月，过早卖出错过后续涨幅
- 缓解：
  - STABLE型使用宽松参数（回撤15%才止盈）
  - 分段止盈（不一次性清仓）
  - 剩余仓位继续参与趋势

**止损过紧风险**：
- 问题：10%止损 × 3倍杠杆 = 价格波动3.3%即触发
- 缓解：
  - STABLE型止损12%（价格波动4%）
  - 可考虑根据实际测试调整

**波动率误判风险**：
- 问题：币种波动率会变化，买入时判断可能不准
- 缓解：
  - 使用30天滚动数据，较为稳定
  - 如需要可添加动态重新分类机制

### 5.2 适用范围

**适用**：
- 趋势型策略
- 中长期持仓（数天到数周）
- 使用杠杆的情况

**不适用**：
- 日内高频交易
- 纯套利策略
- 极短线策略（<1天）

### 9.2 术语表

- **波动率（Volatility）**: 价格波动的幅度和频率
- **ATR（Average True Range）**: 平均真实波幅，衡量波动性的指标
- **快速锁定（Quick Profit Lock）**: 高波动币种在达到一定盈利时立即部分卖出
- **分段止盈（Partial Exit）**: 分多次部分卖出，而非一次性清仓
- **移动止损（Trailing Stop）**: 随价格上涨而上移的止损线
